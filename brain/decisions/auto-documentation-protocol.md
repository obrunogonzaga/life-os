# Auto-Documentation Protocol

## Contexto

Chat context é temporário. Documentos são a memória real.

O Claw (AI partner) deve ativamente externalizar conhecimento importante para o life-os, em vez de depender de contexto de conversa.

## Decisão

A partir de 2026-01-30, Claw opera com as seguintes regras:

### Responsabilidades

1. **Monitorar conversas** para:
   - Conceitos importantes
   - Decisões
   - Ideias recorrentes
   - Modelos mentais
   - Frameworks
   - Projetos de longo prazo

2. **Converter insights em documentos** no `/brain`

### Regras

- **Documentos > Memória interna** — Nunca assumir que contexto será lembrado
- **Criar ou atualizar, nunca duplicar** — Se existe, evoluir
- **Daily journal obrigatório** — Resumo diário de cada interação

### Tipos de documento

| Tipo | Uso |
|------|-----|
| `concept` | Ideias, modelos mentais, frameworks |
| `decision` | Decisões importantes com contexto |
| `journal` | Resumo diário |
| `note` | Anotações gerais |
| `reference` | Materiais externos úteis |

### Triggers para documentação

- Importância explícita ("isso importa", "lembra disso")
- Discussões repetidas entre dias
- Decisões estratégicas, arquiteturais ou de negócio
- Modelos mentais, processos, frameworks

## Consequências

- Claw tem memória durável e navegável
- Conhecimento é reutilizável pelo Bruno futuro
- Menos dependência de contexto de chat
- Base de conhecimento cresce organicamente
